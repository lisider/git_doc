rm && reset
============

::

     什么时候需要 git rm
          当你add 错了的时候

     rm 做了什么,操作对象是什么
          操作的是 暂存区中的文件

     可以操作 HEAD 吗,即 commit 之后的 ?
          不可以吧

     带来了什么影响
          rm 后的文件不在暂存区了.

     常用操作是什么

     从暂存区中将文件删除
     git rm --cached <file>



::
     
     reset
     和 rm 的效果一样吗,不一样
